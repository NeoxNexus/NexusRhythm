## 6. 发现的问题

### 🔴 必须修复（BLOCKING）
- `block-debt-commits.sh` 最初通过 grep 原始 stdin 文本判断 `git commit|git push`，对 JSON 结构依赖过弱，容易误判或漏判。已修复为解析 `tool_input.command` 后再做命令判断。

### 🟡 建议改进（NON-BLOCKING）
- 为 `.claude/hooks/` 和 `/review` 建立最小 smoke tests，避免后续文档或 hook 语义再次漂移
- 继续清理 `GUIDE` / `HANDBOOK` 中可能残留的旧示例，并统一为“Claude Code”而非“Claude Code CLI”表述
- 把 Phase 0 结束仪式进一步自动化，减少手工同步 `ROADMAP` 与产出文档的可能偏差

### 🟢 Nits（可选）
- 后续可以给 `change-auditor` 增加更严格的输出模板，便于多阶段复用
